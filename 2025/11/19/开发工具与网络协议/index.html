一、Git 版本控制

1.1 什么是版本控制？

版本控制系统 (VCS) 是一种记录一个或多个文件内容变化的系统，以便将来可以查阅、恢复到特定版本。

1.2 集中式 vs 分布式

特性 集中式 (SVN) 分布式 (Git)
存储方式 单一中央服务器 每个开发者本地都有完整仓库
网络依赖 强依赖网络 大部分操作可离线完成
速度 较慢 极快
安全性 服务器故障可能导致数据丢失 任何本地仓库都可作为备份

1.3 Git 的工作区域

区域 说明
工作区 当前在电脑上看到和编辑的项目文件
暂存区 临时区域，存放准备提交的文件快照
本地仓库 .git 目录，永久存储提交快照和完整历史
远程仓库 托管在服务器上的仓库，用于团队共享

五、HTTP 协议

5.1 HTTP 请求方法

方法 说明 幂等性 安全性
GET 获取资源 ✅ ✅
HEAD 获取响应头 ✅ ✅
POST 提交数据 ❌ ❌
PUT 更新资源 ✅ ❌
DELETE 删除资源 ✅ ❌
PATCH 部分更新 ❌ ❌
OPTIONS 获取支持的方法 ✅ ✅

5.2 请求报文结构

组成部分：

  1. 请求行：方法 + URL + 协议版本
  2. 请求头：键值对，描述请求信息
  3. 空行：分隔头部和主体
  4. 请求体：实际传输的数据

5.3 响应报文结构

5.4 常用请求头

请求头 说明
Content-Type 请求体类型
Authorization 认证信息
User-Agent 客户端信息
Accept 期望的响应类型
Cookie 携带的 Cookie
Referer 来源页面
Origin 请求来源（CORS）

5.5 常用响应头

响应头 说明
Content-Type 响应体类型
Set-Cookie 设置 Cookie
Location 重定向地址
Cache-Control 缓存控制
Access-Control-Allow-Origin CORS 允许的来源
X-Frame-Options 防止点击劫持

六、HTTP 状态码

6.1 1xx 信息提示

状态码 含义 说明
100 Continue 继续发送请求
101 Switching Protocols 切换协议

6.2 2xx 成功

状态码 含义 说明
200 OK 请求成功
201 Created 资源创建成功
202 Accepted 请求已接受，但未处理
204 No Content 成功但无内容
206 Partial Content 部分内容（断点续传）

6.3 3xx 重定向

状态码 含义 说明
301 Moved Permanently 永久重定向
302 Found 临时重定向
304 Not Modified 资源未修改（使用缓存）
307 Temporary Redirect 临时重定向（保持方法）
308 Permanent Redirect 永久重定向（保持方法）

6.4 4xx 客户端错误

状态码 含义 说明
400 Bad Request 请求语法错误
401 Unauthorized 未授权（需要登录）
403 Forbidden 禁止访问（权限不足）
404 Not Found 资源不存在
405 Method Not Allowed 方法不允许
408 Request Timeout 请求超时
409 Conflict 请求冲突
415 Unsupported Media Type 不支持的媒体类型
429 Too Many Requests 请求过多（限流）

6.5 5xx 服务器错误

状态码 含义 说明
500 Internal Server Error 服务器内部错误
501 Not Implemented 功能未实现
502 Bad Gateway 网关错误
503 Service Unavailable 服务不可用
504 Gateway Timeout 网关超时
