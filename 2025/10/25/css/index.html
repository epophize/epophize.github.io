📘 CSS 核心知识点总结

一、 前端三层架构

层次 技术 作用
结构层 HTML 搭建页面骨架与内容结构
表现层 CSS 美化页面，控制布局与视觉效果
行为层 JavaScript 实现交互逻辑与前后端数据通信

二、 CSS 概述

  1. 全称：Cascading Style Sheets（层叠样式表）
  2. 作用：定义如何显示 HTML 元素
  3. 发展意义：HTML 4.0 引入，核心目的是内容与表现分离
  4. 特点：
    • 样式通常存储在外部 .css 文件中，可极大提高开发效率与维护性
    • 多个样式定义可按规则层叠为一个最终样式
    • 通配符选择器 * 可选取页面所有元素

三、 CSS 引入方式

方式 实现方法 特点
外部样式表（推荐） <link rel="stylesheet" href="style.css"> 写在 <head> 缓存复用，最常用
内部样式表 <head> 中使用 <style> 标签包裹 CSS 代码 适用于单页特定样式
内联样式 标签内直接写 style="属性:值;" 优先级高，但难维护，尽量少用

四、 选择器与优先级

🔹 基础选择器

类型 语法 说明
标签选择器 p, span 选中所有同名标签
类选择器 .className 可复用，命名不可为中文或纯数字
ID 选择器 #idName 页面唯一，命名不可为中文或纯数字

🔹 组合选择器

类型 符号 作用范围
后代选择器 空格 选中所有后代元素（嵌套层级不限）
子代选择器 > 仅选中直接子元素
并集选择器 , 同时选中多个不同类型的元素

🔹 优先级（权重）规则

  1. 同类型选择器：后写的样式覆盖先写的（就近原则）
  2. 不同类型权重（理论值）：
    !important (10000) > 内联样式 (1000) > ID选择器 (100) > Class选择器 (10) > 标签选择器 (1) > 通配符 (0.1)
  3. !important：强制提升优先级。仅在代码庞大、常规权重无法覆盖时使用，滥用会导致维护困难。

五、 常用样式属性

📝 1. 文本样式 (text-*)

💡 注意：所有文本属性默认具有继承性

属性 常用值 说明
color red, #ff0000, rgb(255,0,0) 文本颜色（支持名称/十六进制/rgb）
line-height 22px, 1.5, 150% 行高（数值会与当前字体大小相乘）
text-align left, center, right 文本水平对齐
vertical-align top, middle, bottom 元素垂直对齐，常用于图文混排
text-decoration none, underline, overline, line-through 文本修饰线（去下划线常用 none）
text-indent 2em, 32px 首行缩进
text-shadow 5px 5px 5px #FF0000 阴影：水平 垂直 模糊 颜色
letter-spacing 2px 字符间距（支持负值）
word-spacing 5px, normal 单词间距（以空格分隔）
text-transform capitalize, uppercase, lowercase 英文大小写转换

🔗 2. 链接伪类 (LVHA 原则)

⚠️ 书写顺序建议固定为 :link → :visited → :hover → :active，避免样式冲突。

🔤 3. 字体样式 (font-*)

属性 常用值 说明
font-family "Microsoft YaHei", sans-serif 字体系列，建议多写备用字体
font-size 16px, 1.2em, larger, smaller 字体大小
font-weight normal(400), bold(700), 100~900 字体粗细，≤400 常规，≥600 加粗

📋 4. 列表样式 (list-style)

属性 常用值 说明
list-style-type disc, circle, square, decimal, none 列表项标记类型
list-style-position outside(默认), inside 标记位置（文本外/内）
list-style-image url('icon.png') 自定义标记图片
简写 list-style: type position image; 常用：list-style: none; 清除默认样式

🖼️ 5. 背景样式 (background-*)

属性 常用值 说明
background-color 颜色值 / transparent 背景色，默认透明
background-image url('路径') 背景图
background-repeat repeat, repeat-x, repeat-y, no-repeat 背景图重复方式
background-position center center, 0 0, 50% 50% 背景图起始位置
background-size cover, contain, 100px 200px, 100% cover短边铺满，contain长边铺满
background-attachment scroll(默认), fixed 背景随页面滚动或固定
简写 background: color image position/size repeat attachment; 例：background: #fff url(bg.jpg) center/cover no-repeat fixed;

六、 盒模型 (Box Model)

📦 四大组成部分

  • Content：内容区域（文本、图片等）
  • Padding（内边距）：内容到边框的距离，透明，可控制
  • Border（边框）：围绕内边距的边界
  • Margin（外边距）：盒子与其他元素之间的距离，透明

📐 盒模型计算规则

模型 box-sizing 值 实际宽度计算 实际高度计算
标准盒模型 content-box（默认） width + 左右padding + 左右border height + 上下padding + 上下border
怪异/IE盒模型 border-box width（已包含 padding 和 border） height

🔧 常用属性

  • 内边距/外边距简写规则（同 padding 与 margin）：
    1值：上下左右｜2值：上下｜左右｜3值：上｜左右｜下｜4值：上｜右｜下｜左（顺时针）
  • 边框简写：border: 1px solid #ccc;（style 属性必填）
  • 轮廓 outline：绘制于边框外围，不占布局空间，语法同 border。常用于表单 focus 状态。

七、 尺寸与单位

属性 说明
width / height 设置宽高
max-width / min-width 限制最大/最小宽度
max-height / min-height 限制最大/最小高度

📏 常用单位

  • px：绝对单位（像素）
  • em：相对单位（相对于父元素字体大小）
  • rem：相对单位（相对于根元素 <html> 字体大小，默认 1rem = 16px）
  • %：相对单位（相对于父容器对应维度）

八、 溢出与光标控制

📦 overflow 内容溢出处理

值 效果
visible（默认） 不裁剪，溢出内容可见
hidden 裁剪溢出内容，不提供滚动条
scroll 始终显示滚动条（即使不溢出）
auto 仅在溢出时显示滚动条

可拆分为 overflow-x 和 overflow-y 单独控制。

🖱️ cursor 鼠标指针样式

常用值：default（箭头）, pointer（小手，常用于按钮/链接）, text（文本输入）, wait（忙/加载）, crosshair（十字）, url('custom.cur'), auto（自定义）

九、 元素显示与可见性

属性 值 特点
display block 块级：独占一行，可设宽高/边距
inline 行内：同行显示，宽高/上下边距无效
inline-block 行内块：同行显示，可设宽高/边距
none 不显示且不占空间（脱离渲染树）
visibility hidden 不可见，但保留占位空间（相当于透明）
opacity 0 ~ 1 透明度，0全透明，1不透明。子元素会继承透明度

十、 浮动 (Float) 与 BFC

🌊 float 特性

  • left / right：元素向左/右浮动，脱离标准文档流，不占位置
  • 浮动元素会尽可能向左/右移动，直到遇到父级边界或其他浮动元素
  • 浮动元素后的块级元素会环绕浮动元素，但前的元素不受影响

🧹 clear 清除浮动

  • left / right / both：禁止元素左侧/右侧/两侧出现浮动元素

📉 父级高度塌陷 & 清除浮动方案

浮动导致父元素高度为 0（塌陷），常用解决：

  1. 末尾加空块级元素：<div style="clear:both;"></div>（不推荐，冗余标签）
  2. 父级设置：overflow: hidden;（触发 BFC）
  3. 父级固定高度：height: xxx;（不灵活）

十一、 定位 (Position)

值 参照物 是否脱离文档流 占位 说明
static 无 ❌ ✅ 默认值，不受 top/left 影响
relative 自身原位置 ❌ ✅ 相对偏移，原位置保留
absolute 最近的非 static 父级 ✅ ❌ 找不到则相对 <body> 定位
fixed 浏览器可视窗口 ✅ ❌ 滚动页面时位置固定
sticky 滚动容器 混合 ✅ 阈值内正常，超出后变为 fixed（吸顶）

📐 偏移与层级

  • top, right, bottom, left：定义定位元素的偏移量
  • z-index：控制 Z 轴层叠顺序（数值越大越靠前）。仅对非 static 定位元素有效，支持负值。

十二、 实用技巧与注意事项

🛠️ margin-top 塌陷（外边距重叠）

父子块元素垂直方向 margin 会合并，导致子元素 margin-top 作用于父元素。
解决方法：

  1. 父元素添加 padding-top
  2. 父元素添加 border-top
  3. 父元素设置 overflow: hidden;（触发 BFC）
  4. 父元素设置 display: flow-root;（现代推荐）

✨ Hover 效果实现步骤

  1. 正常布局父容器
  2. 将 hover 显示的内容放入父容器内
  3. 父容器 position: relative;，子内容 position: absolute; top: 0; left: 0;
  4. 编写子内容 hover 样式
  5. 默认隐藏子内容（display: none 或 opacity: 0），父级 :hover 时显示

🧼 CSS 初始化 (Reset) 的作用

  1. 解决浏览器兼容问题：不同浏览器对标签默认样式（margin/padding/字体等）定义不同
  2. 统一代码规范：清除默认样式，保证多端/多浏览器表现一致，提高开发效率

💡 学习建议：CSS 的核心在于理解盒模型、文档流、定位与层叠上下文。多结合浏览器开发者工具（F12）调试 computed 样式与布局面板，能大幅提升实战效率。
