一、Vue 简介与核心概念

1.1 什么是 Vue？

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。

  • 核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合
  • Vue 借鉴 React 和 Angular 的部分代码设计，并提高了易用性和轻量化

1.2 MVVM 设计模式

MVVM：Model-View-ViewModel（模型 - 视图 - 视图模型）

概念 说明
数据模型 (Model) 构成页面内容的相关数据（前端定义的数据、后端传递的数据）
视图 (View) 呈现给开发者和用户查看的数据展示页面
视图模型控制器 (ViewModel) MVVM 的核心思想，连接 View 和 Model 的桥梁

💡 提示：前端实现 MVVM 设计思想的框架（Vue、React、Angular、微信小程序等），目的都是为了高度封装 ViewModel 的交互过程，让开发者只关心页面构成和数据构成，无需花费大量时间关心数据和页面的状态关系。

1.3 Vue 2 vs Vue 3 关键对比

特性 Vue 2 Vue 3
响应式原理 Object.defineProperty Proxy（支持 Map/Set、数组索引监听）
API 风格 选项式 API（Options API） 组合式 API（Composition API）为主，保留选项式 API
Tree-shaking 不支持 支持（按需引入，减小包体积）
TypeScript 支持 需额外配置 原生支持（源码用 TS 编写）
模块加载规则 基于 Common.js 的动态模块 使用浏览器原生 ESM 模块规则
工程框架 Webpack Vite（按需加载）

2.3 基础指令

Vue 指令以 v-名称 结构定义，只能用于 HTML 容器的标签属性上。

指令 取值 功能 示例
v-text string 更新元素的 textContent <span v-text="msg"></span>
v-html string 更新元素的 innerHTML <div v-html="html"></div>
v-pre 不需要 跳过编译过程 <span v-pre>{{ 直接显示 }}</span>
v-once 不需要 只执行一次 <span v-once>{{msg}}</span>
v-cloak 不需要 配合 CSS 隐藏未编译模板 [v-cloak]{display:none}
v-show any 切换 display 属性 <p v-show="flag"></p>
v-if/v-else any 条件渲染 <div v-if="type==='A'">A</div>
v-for Array/Object/number/string 列表渲染 <div v-for="item in items">
v-on (@) Function 事件绑定 <button @click="handle">
v-bind (:) any 属性绑定 <img :src="imageUrl">
v-model any 双向数据绑定 <input v-model="text">

四、Vue 3 新特性

4.1 响应式原理升级

Vue 2: Object.defineProperty

  • 无法检测对象属性的添加或删除
  • 无法监控数组索引和长度的变化

Vue 3: Proxy

  • 支持 Map、Set、WeakMap、WeakSet
  • 支持数组索引监听
  • 性能提升 1.3~2 倍

  • Teleport - 传送门组件
  • Suspense - 异步组件加载状态

6.2 组件命名规范

  • 使用多单词组件名，避免与 HTML 元素冲突
  • 文件名与组件名保持一致
  • 使用 PascalCase 或 kebab-case

6.3 性能优化

  1. 使用 v-show 替代 v-if 用于频繁切换的场景
  2. 使用 key 属性 优化 v-for 列表渲染
  3. 使用 Object.freeze() 冻结不需要响应式的大型数据
  4. 使用异步组件 按需加载大型组件
  5. 使用 v-once 标记不需要的动态内容
